# 制作パイプライン設計

## 全体フロー

この書庫は、資料から完成イラストまでを一連の制作パイプラインとして扱う。

```text
使える資料
  ↓
テストプロンプト作成
  ↓
生成テスト
  ↓
レビュー
  ↓
改善
  ↓
書庫入り
  ↓
キャラ・世界観・補正と組み合わせ
  ↓
完成イラスト生成
  ↓
アルバム保存
```

## 1. 資料

対象は Deep Research に限定しない。

候補:

- Deep Research
- 書籍メモ
- 動画講座
- SNS の技法メモ
- 既存作例の分析
- 過去の成功プロンプト
- 失敗から得た改善メモ

資料はそのまま書庫に入れず、まず「何の表現に使えるか」を抽出する。

## 2. テストプロンプト

資料から得た知見を、仮の生成用プロンプトに変換する。

この段階では完成度よりも、検証したい表現が明確であることを重視する。

必要項目:

- 目的
- 参照資料
- 検証したい表現
- 仮プロンプト
- 想定される失敗
- テスト日
- ステータス

ステータス例:

```yaml
status: untested
status: testing
status: needs_revision
status: candidate
status: rejected
```

## 3. レビュー

生成結果を見て、書庫入りできるかを確認する。

レビュー観点:

- 目的の表現が一目で伝わるか
- 主役、動き、構図が読み取りやすいか
- 差し替え可能な構造になっているか
- 専用キャラと組み合わせやすいか
- 破綻しやすい要素が把握できているか
- デフォルト補正が必要か
- 作例と所見が残っているか
- 再利用価値があるか

合格したものだけを書庫入りにする。

## 4. プロンプト書庫

書庫には、レビュー済みのプロンプトを保存する。

書庫プロンプトは「表現の核」に集中させ、専用キャラや品質補正を混ぜ込みすぎない。

保存項目:

- ID
- タイトル
- 目的
- 本文
- 差し替え欄
- 推奨アセット
- 推奨デフォルト補正
- 参照カード・モジュール
- 作例
- テスト所見
- ステータス

## 5. アセット

アセットは、書庫プロンプトと組み合わせる素材として管理する。

アセット種別:

- 専用キャラ
- 衣装
- 背景・舞台
- 世界観
- ライティング
- 色調
- 構図
- ポーズ
- NG 指定
- 生成済み作例
- 完成イラスト

データ上は次のように分類する。

```yaml
asset_type: character
asset_type: costume
asset_type: background
asset_type: world
asset_type: lighting
asset_type: composition
asset_type: negative
asset_type: finished_image
```

## 6. デフォルト補正

崩れやすい部分は、書庫プロンプト本文に常時埋め込まず、オンオフ可能な補助レイヤーとして扱う。

主な補正カテゴリ:

- 手・指
- 姿勢・骨格
- 顔・目
- 衣装破綻
- 構図安定
- 画像品質
- NG 指定
- アクション向け補正
- ポートレート向け補正

例:

```json
{
  "id": "default-hands-stability",
  "label": "手・指を安定させる",
  "category": "anatomy",
  "positive": [
    "natural hands",
    "clear finger separation",
    "visible thumb placement"
  ],
  "negative": [
    "extra fingers",
    "missing fingers",
    "fused fingers",
    "broken hands"
  ],
  "recommended_for": [
    "portrait",
    "full_body",
    "action"
  ]
}
```

プリセット例:

- 軽め補正
- 標準補正
- 厳密補正
- アクション向け
- ポートレート向け

## 7. 生成

生成時は、以下を組み合わせて最終生成プロンプトを作る。

```text
書庫プロンプト
  + 選択アセット
  + デフォルト補正
  + NG 指定
  + 用途別出力設定
```

UI では、チェックボックスやプリセットでオンオフできるようにする。

例:

```text
[✓] 手・指を安定させる
[✓] 姿勢・骨格を安定させる
[ ] 顔の崩れを防ぐ
[✓] 衣装破綻を防ぐ
[ ] 強めの NG 指定を入れる
[ ] アクション向け補正
```

## 8. アルバム

完成イラストはアルバムとして保存する。

保存項目:

- 完成イラスト
- 使用した書庫プロンプト
- 組み合わせたアセット
- 有効化したデフォルト補正
- 最終生成プロンプト
- 生成日
- 使用モデル・ツール
- 所見
- 再利用タグ

アルバムは単なる画像一覧ではなく、成功した制作レシピの履歴として扱う。
