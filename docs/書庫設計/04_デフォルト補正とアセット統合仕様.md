# デフォルト補正とアセット統合仕様

## 目的

デフォルト補正は、書庫プロンプトの本文に固定で埋め込まない。

まずテストプロンプトとして検証し、使う価値があるものだけをオン/オフ可能な補助レイヤーとして扱う。

書庫では、ユーザーが以下を選ぶことで最終生成プロンプトを統合できるようにする。

- デフォルト補正を使うか
- 参考画像が必要か
- どの参考画像を使うか
- キャラクター専用プロンプトをアセットから追加するか
- 衣装、背景、世界観、NG指定を追加するか

## 基本仕様

UIでは、各補正にチェックボックスを持たせる。

オンにした補正だけが最終生成プロンプトに表示される。

オフにした補正は、プレビューにもコピー結果にも出さない。

```text
[✓] 手・指を安定させる
[✓] 姿勢・骨格を安定させる
[✓] 主役のシルエットを読みやすくする
[ ] 顔・目の崩れを防ぐ
[✓] 衣装破綻を防ぐ
[ ] アクションの重心と勢いを保つ
[ ] 強めのNG指定を入れる
```

## 初期オンにする補正

### 手・指

多くの人物イラストで破綻しやすく、入れても表現への副作用が比較的小さい。

初期オン。

ただし、手が見えない構図ではオフでよい。

### 姿勢・骨格

全身、アクション、ポーズ指定で破綻しやすい。

初期オン。

ただし、顔アップや抽象表現ではオフでよい。

### 主役のシルエット

書庫プロンプトの再利用性を高めるため、主役が読めることを優先する。

初期オン。

ただし、群衆、ホラー、ミステリーなど、あえて隠す演出ではオフにする。

### 衣装破綻

専用キャラやアルバム用途では、衣装の一貫性が重要になる。

初期オン。

ただし、ラフな探索や衣装指定が少ない場合はオフでもよい。

## 初期オフにする補正

### 顔・目

顔が主役の時は重要だが、アクションや遠景では不要なことがある。

初期オフ。

ポートレート、上半身、表情重視の時だけオンにする。

### アクション重心

アクション系には有効だが、静止画やポートレートでは不要。

初期オフ。

戦闘、スポーツ、ダンス、乗り物、追跡ではオンにする。

### 強めのNG指定

破綻を抑える効果はあるが、モデルによっては画面が硬くなる。

初期オフ。

破綻が連続する時、アルバム品質を優先する時だけオンにする。

## データ構造

デフォルト補正は `defaults/default-corrections.json` に置く。

主な項目:

```json
{
  "id": "DEF-HANDS-001",
  "label": "手・指を安定させる",
  "category": "anatomy",
  "default_state": "on",
  "initial_checked": true,
  "positive": ["natural hands"],
  "negative": ["extra fingers"],
  "turn_on_when": ["hands are visible"],
  "turn_off_when": ["hands are fully hidden"],
  "test_prompt_id": "TPR-DEFAULT-CORRECTION-SUITE-001"
}
```

## テストプロンプト

デフォルト補正自体は、`prompts/testing/TPR-DEFAULT-CORRECTION-SUITE-001.md` で検証する。

比較パターン:

- 補正なし
- 標準補正オン
- アクション補正オン
- 厳密補正オン

確認すること:

- 破綻が減るか
- 表現の勢いが落ちないか
- キャラの個性が薄くならないか
- 強いNG指定の副作用がないか

## 書庫プロンプトとの統合

最終生成プロンプトは、次の順番で組み立てる。

```text
1. 書庫プロンプト
2. キャラクターアセット
3. 衣装アセット
4. 背景・世界観アセット
5. 参考画像メモ
6. オンにしたデフォルト補正
7. 用途別指定
8. オンにしたNegative補正
```

## 参考画像の扱い

参考画像は、すべてのプロンプトに必須ではない。

書庫プロンプト側では、次を選べるようにする。

```text
[ ] 参考画像を使う
```

オンにした場合だけ、画像選択欄を表示する。

```text
参考画像:
  [キャラ正面]
  [衣装設定]
  [表情資料]
  [ポーズ資料]
  [背景資料]
```

選んだ画像は、最終プロンプト本文に画像そのものを埋め込むのではなく、生成依頼用のメモとして表示する。

例:

```text
Reference images:
- Use character_front_001 as the primary character identity reference.
- Use costume_black_coat_001 for outfit consistency.
- Use pose_sprint_low_angle_001 for body direction and camera angle.
```

## キャラクター専用プロンプトの扱い

キャラクター専用プロンプトは、書庫プロンプト本文とは分離する。

UIでは次のように選ぶ。

```text
[ ] キャラクターアセットを追加する
```

オンにした場合だけ、キャラクター選択欄を表示する。

```text
キャラクター:
  [黒髪の少女剣士]
  [白髪の魔法使い]
  [専用マスコット]
```

キャラクターアセットには以下を持たせる。

- 名前
- 外見プロンプト
- 固有衣装
- 性格・雰囲気
- NG指定
- 参考画像
- 使用上の注意

## 最終プロンプトの例

```text
Base prompt:
高速の近接戦。低いカメラ位置、対角線構図、すれ違い直前の一瞬、雨の路地、水しぶき。

Character asset:
黒髪の少女剣士。長い黒髪、黒いロングコート、細身の刀、冷静だが緊張感のある表情。

Reference images:
- Use character_front_001 as the primary character identity reference.
- Use costume_black_coat_001 for outfit consistency.

Default corrections:
natural hands, clear finger separation, balanced anatomy, clear readable silhouette, consistent clothing design

Negative prompt:
extra fingers, missing fingers, broken hands, twisted limbs, melted clothing, unreadable pose
```

## 書庫入りの考え方

書庫プロンプトは、表現の核として保つ。

デフォルト補正、参考画像、キャラクター専用プロンプトは、生成時に足す。

これにより、書庫プロンプトが特定キャラに閉じず、専用キャラ制作にも流用できる。
