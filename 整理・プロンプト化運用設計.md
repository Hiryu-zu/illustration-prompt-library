# Deep Research をプロンプト集へ育てる運用設計

## 結論

`inbox` は未加工資料の投入口に限定し、以後の情報を次の4層で管理する。

1. **Source（原資料）** — Deep Research の全文と出典を保存する
2. **Card（知識カード）** — 一つの原理を、制作で使える最小単位へ分解する
3. **Module（プロンプト部品）** — Card を生成AIが解釈できる視覚記述へ翻訳する
4. **Prompt（完成プロンプト）** — 複数 Module を目的別に組み、生成検証したものを公開する

原資料から完成プロンプトへ直接コピーしないことが、この仕組みの中心である。原理、表現、完成品を分けることで、同じ研究をアクション、心理効果、SNS、キャラクター設計などへ再利用できる。

```text
inbox（未処理）
  ↓ 受付・台帳登録
sources（原資料）
  ↓ 一知見一枚に分解
cards（知識カード）
  ↓ 視覚的な指示へ翻訳
modules（プロンプト部品）
  ↓ 目的に応じて合成・生成テスト
prompts（完成プロンプト）
  ↓ 公開
プロンプトビューアー / プロンプト書庫
```

## 推奨フォルダ

```text
inbox/                 新しく得た未処理資料。原文は変更しない
catalog/               資料台帳、カード索引、タグ辞書
sources/               受付済み原資料。出典と取得日を保持
cards/                 一知見一枚の知識カード
modules/               組み替え可能なプロンプト部品
prompts/
  draft/               未検証の完成プロンプト
  tested/              生成テストを通したもの
  published/           HTMLへ掲載したもの
templates/             各層の記入雛形
outputs/               作例画像と生成条件
archive/               旧版・重複・廃止項目
```

既存のHTMLは当面そのまま維持し、`prompts/published` を正本、HTMLを閲覧用の生成物とする。将来は公開済みファイルからHTML内のJSONを生成すれば二重編集を避けられる。

## 分類はフォルダではなくタグで行う

一つの知見は複数の用途を持つため、深いフォルダ階層にはしない。各 Card / Module に次の軸を付ける。

| 軸 | 例 | 用途 |
|---|---|---|
| `goal` | 迫力、ワクワク、癒し、恐怖、愛着、停止率 | 欲しい効果から逆引き |
| `control` | 主体、ポーズ、構図、カメラ、光、色、背景、時間、質感 | 画面のどこを操作するか |
| `scene` | 近接戦、旅立ち、日常、恋愛、深海、SNS | 題材から絞り込み |
| `phase` | 溜め、直前、命中、振り抜き、余韻 | 時間相の指定 |
| `evidence` | strong、moderate、tentative、practice | 根拠の強さ |
| `status` | extracted、draft、tested、published、deprecated | 成熟度 |
| `model` | generic、Midjourney、SDXL、Flux、ImageGen | 方言・最適化先 |

タグは増やしすぎず、`catalog/タグ辞書.md` に登録した語だけを使う。特に `goal` と `control` は必須とする。

## 1件を処理する手順

### 1. 受付

- 原文を `inbox` に保存し、内容は書き換えない
- `catalog/調査資料台帳.json` に Source ID、題名、取得日、対象範囲を登録する
- ファイル名が曖昧でも、原本名は維持して台帳側で意味を与える

### 2. 選別

全文を要約するのではなく、次の条件を満たす箇所だけ Card 候補にする。

- 絵のどの要素を変えるか説明できる
- 期待する視覚効果または心理効果が明確
- 失敗条件・副作用・強度調整のいずれかが分かる
- 他の題材にも転用できる

固有の事例、重複説明、創作へ翻訳できない背景説明は Source に残し、Card 化しない。

### 3. Card 化

一枚に一つの因果だけを書く。

> 例：「選択的モーションブラー」＝顔と軸を鮮明に残し、末端と背景だけを流すと、可読性を保ちながら速度が伝わる。

Card には「原理」「操作」「効く理由」「失敗」「強度」「出典」を持たせる。研究上の主張と、そこから導いた制作上の推論は明記して分ける。

### 4. Module 化

抽象語を、画面に観測できる記述へ変換する。

| 抽象語だけの弱い指定 | 観測可能な指定 |
|---|---|
| 迫力のある絵 | 低い重心、手前へ迫る拳、対角線構図、接地点から同方向へ飛ぶ破片 |
| ワクワクする場面 | 背中越しの視点、奥へ続く道、半開きの門、遠景に一つの目的物 |
| スピード感 | 顔と軸脚は鮮明、髪先・武器先端・背景だけを進行方向へ流す |

Module は主に次のスロットへ分ける。

```text
subject      誰・何を描くか
action       動作と時間相
body         重心、支持、ひねり、視線
composition  配置、視線導線、余白
camera       画角、距離、角度、パース
lighting     光源、明暗、焦点
color        色調、補色、差し色
environment  背景、小物、反作用、物語の痕跡
motion       慣性、残像、ブラー、飛散方向
constraints  鮮明に残す箇所、禁止事項、上限
```

単語の羅列ではなく、「何が」「どこへ」「どの程度」を含む短文を基本にする。

### 5. 完成 Prompt の合成

最初に目的を一つ決め、必要な Module だけを選ぶ。

```text
目的 → 主役 → 時間相 → 身体 → 構図/カメラ → 光/色
     → 背景の証拠 → 動き/反作用 → 制約 → ネガティブ
```

一枚で複数の心理効果を狙う場合も、主目的1つ、副目的は最大2つまでとする。すべてを盛ると、焦点と検証可能性が失われる。

### 6. 生成テスト

完成 Prompt は一度生成しただけでは `tested` にしない。最低3出力を見て次を記録する。

- 主役と動作が一秒で読めるか
- 狙った効果が、説明なしでも感じられるか
- 重要な Module が実際に反映されたか
- 破綻または副作用は何か
- 削っても結果が変わらない冗長語は何か
- 使用モデル、日付、主要設定、作例画像の場所

成功条件を満たしたら `tested`、HTMLへ載せたら `published` にする。モデル更新で結果が変わったものは再試験する。

## ID と命名

表示名は日本語、IDはリンク切れを防ぐため半角英数で固定する。

```text
Source: SRC-20260625-ACTION
Card:   CARD-MOTION-SELECTIVE-BLUR
Module: MOD-MOTION-SELECTIVE-BLUR-JA
Prompt: PRM-ACTION-HEAVY-PUNCH-001
```

Card と Module のファイル名はIDに合わせる。題名やタグが変わってもIDは変更しない。

## 今回の3資料の切り分け方

| Source | 主な Card 群 | 最初に作る価値が高い Module |
|---|---|---|
| アクション表現 | 力の方向、重心、時間相、慣性、選択的ブラー、反作用 | アクションの身体、速度、衝撃、カメラ |
| 心理に作用する絵 | 図地分離、視線、未完、感情、愛着、畏怖、ノスタルジア | 感情別の構図、焦点、物語の痕跡 |
| ワクワク | 前向きな予期、情報ギャップ、可達性、参加感、逆U字 | 旅立ち、探索、変身直前、共闘、秘密 |

優先して統合するとよい交点は次の3つ。

1. **アクション × ワクワク** — 命中後より「踏み込む直前」、完成形より「覚醒直前」
2. **心理 × アクション** — 顔・命中点・武器のどこを主焦点にするか決め、反作用を同じ力ベクトルへ揃える
3. **心理 × ワクワク** — 一読目の明快な主役と、二読目で見つかる未回収の手がかりを共存させる

## 品質基準

公開する項目は、次の条件をすべて満たす。

- Source まで逆引きできる
- 抽象的な感情語だけに頼っていない
- 主目的と、操作する画面要素が明確
- 「足すこと」だけでなく、鮮明に残すもの・抑えるものも指定している
- 失敗条件または使わない条件がある
- 生成モデルと作例を伴う
- 既存項目との差分を一文で説明できる

## 最小運用ルール

- `inbox` は週1回または5件たまった時点で処理する
- 1 Source から最初に作る Card は5〜12枚に絞る
- 1 Card から Module を無理に作らない。画面操作へ翻訳できるものだけにする
- 完成 Prompt は複製せず、共通部分を Module として参照する
- タグ追加、公開、廃止は台帳へ反映する
- 原本、引用、作例画像は削除しない

## 次の実装段階

まず今回の3資料から、各5枚ずつ計15枚の Card を作る。その後、重複を統合して10前後の Module にし、次の3本を試作すると全工程を検証しやすい。

1. 「旅立ち直前のワクワク」
2. 「重量感のある決定的な拳撃」
3. 「一読目は明快、二読目で謎が残るキャラクター絵」

この小さな一巡を先に行い、タグと雛形を調整してから大量変換する。
