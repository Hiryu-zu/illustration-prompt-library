# プロンプト書庫 ユーザビリティ設計書

対象: `プロンプト書庫.html`
版: v2 設計（完成版セット＋個別プロンプト選択）
日付: 2026-06-18

---

## 1. 背景と問題提起

現行版は「完成プロンプト」を巻ごとに並べて選ばせる構成だが、提示される本文が次のような一行に留まっており、これだけでは生成プロンプトとして完成していない。

```
masterpiece, best quality, ultra-detailed, 8k, 1girl, solo, upper body, ... bokeh
```

不足しているのは「ネガティブプロンプト」「推奨生成設定（サイズ・サンプラー・CFG等）」、そして「自分の用途に合わせて要素を足し引きする手段」である。本設計はこれらを解消する。

---

## 2. 現状のUX課題（洗い出し）

| # | 課題 | 重大度 | 対応方針 |
|---|------|--------|----------|
| C1 | 完成版が本文一行のみ。ネガ・設定がなく、そのままでは絵が安定しない | 高 | 各頁を「ポジ＋ネガ＋推奨設定」の完成版セットに再構成 |
| C2 | 要素単位の取捨選択ができない（全部入りかコピーしないかの二択） | 高 | 構成要素を個別プロンプト化し、チェックボックスで選択→組み立て→コピー |
| C3 | 戻る動線が弱い。本へ入ると「本棚へ」一箇所のみ、検索・栞からの復帰が不明瞭。ブラウザの戻るも未整備 | 高 | 明示的な「戻る」＋パンくず、hashルーティング整備、ESCで一段戻る |
| C4 | レイアウト崩れ。背表紙の折返し／棚板のズレ、長いタイトル・タグの溢れ、狭幅での詰まり | 高 | 本棚をグリッド化、`min-width:0`徹底、溢れ対策、横スクロール禁止 |
| C5 | コピー対象が複数あるとき主従が不明（何がコピーされるか分かりにくい） | 中 | ボタン文言を対象明示（完成版／選択分／ネガティブ／設定）、主ボタンを強調 |
| C6 | キーボード操作・フォーカス可視・コントラストが未整備 | 中 | 実ボタン/チェックボックス、focus-visible、aria、両テーマでコントラスト確保 |
| C7 | 検索結果・栞・空状態の見せ方が簡素 | 低 | 件数表示・空状態メッセージ・現在地表示を統一 |
| C8 | モバイルでのタップ領域・詰まり | 中 | タップ44px目安、余白調整、要素折返し |

## 3. 設計ゴール

1. **そのまま使える**: 1クリックで「ポジ＋ネガ＋推奨設定」が揃った完成版を写し取れる。
2. **選んで作れる**: 構成要素をチェックで取捨選択し、自分用プロンプトを即組み立て・コピーできる。
3. **迷わない**: 現在地と戻り方が常に明確（パンくず＋戻る＋ESC＋ブラウザ戻る）。
4. **崩れない**: どの画面幅でもレイアウトが破綻せず横スクロールしない。
5. **書庫の趣**: 本棚・明朝・羊皮紙の世界観は維持しつつ、操作性を損なわない。

## 4. 情報設計（IA）と画面遷移

```
本棚（Home / #）
  ├─ 第n巻（Book / #v1）            ← 戻る: 本棚
  │     └─ 頁（Entry: 展開で詳細）
  │            ├─ 完成版セット（ポジ＋ネガ＋設定）＋[完成版を写す]
  │            ├─ 構成要素チェックリスト（個別プロンプト）
  │            ├─ 組み立てプレビュー ＋[選択分を写す]
  │            └─ [ネガを写す] [設定を写す] [栞]
  ├─ お気に入り（#__marks）          ← 戻る: 本棚
  └─ 検索結果（検索入力中）          ← 戻る: 本棚 / 検索クリア
```

ナビゲーション規則:
- ヘッダーは常時固定（書庫名＝Home、検索、本棚へ、昼夜）。
- 本・栞・検索の各画面上部に「← 本棚へ」＋パンくず（例: `本棚 › 第一巻 立ち絵`）。
- `Esc`: 検索中なら検索解除、本の中なら本棚へ（＝一段上がる）。
- ブラウザの戻る/進むは `location.hash` と同期。

## 5. 頁（Entry）の構造仕様 ★中核

各頁は「完成版」と「個別プロンプト」を両立させる。

### 5-1. 完成版セット
- **ポジティブ**: 構成要素テキストを順に結合した完成形。
- **ネガティブ**: 巻ごとの既定ネガ（必要に応じ頁で上書き）。
- **推奨設定**: サイズ／サンプラー／Steps／CFG／Hires.fix の目安。
- 操作: `[完成版を写す]`（ポジ＋ネガ＋設定を整形して一括コピー）。

### 5-2. 構成要素（個別プロンプト）チェックリスト
- 各要素は `ラベル ＋ 英語プロンプト断片 ＋ チェックボックス`。
  例: ☑ 品質 `masterpiece, best quality, ultra-detailed` ／ ☑ 構図 `1girl, solo, upper body` ／ ☑ 髪 `long black hair, hime cut` …
- 既定は全選択（＝完成版ポジと一致）。
- 操作: `[全選択] [全解除]`、各チェックでプレビュー即更新。

### 5-3. 組み立てプレビュー
- 選択中の要素だけを結合した本文をリアルタイム表示。
- 操作: `[選択分を写す]`（チェックした要素のみコピー）。

### 5-4. 補助
- `[ネガを写す]` `[設定を写す]` 個別コピー。
- `[栞]` トグル（localStorage 保存）。

## 6. レイアウト/レスポンシブ方針（C4対策）

- **本棚**: `display:grid; grid-template-columns:repeat(auto-fill,minmax(56px,max-content))` で背表紙を整列。棚板は各行コンテナの擬似要素で安定描画。折返しても棚板がズレない。
- 文字溢れ: 縦書きタイトルは行数制限、タグは `flex-wrap`、本文 `pre` は `white-space:pre-wrap; word-break:break-word`。
- 全コンテナ `min-width:0; max-width:100%`、`main` は `overflow-x:hidden`。
- ブレークポイント: 〜680px でヘッダー・本棚・頁の余白とフォント縮小、ボタンは折返し。
- タップ領域: 主要ボタン高さ ≥ 36–44px。

---

## 7. アクセシビリティ（C6対策）

- 操作要素は実 `<button>` / `<input type="checkbox">` ＋ `<label>`。
- フォーカス可視（`:focus-visible` で枠）。
- 栞は `aria-pressed`、本棚の背表紙は `role="button"` + `aria-label="第一巻 立ち絵を開く"`。
- 昼/夜どちらでも本文・ボタンのコントラスト比 4.5:1 以上を目安。
- キーボード: `/` 検索フォーカス、`Esc` 一段戻る、Tab順は視覚順に一致。

## 8. データスキーマ（実装用）

```js
NEG = { std:"…", real:"…", anime:"…" }          // ネガティブ既定
SET = { portrait:"…832x1216…", land:"…1216x832…" } // 推奨設定既定

VOLUMES = [{
  id, no, title, color, preface,
  neg:"std", set:"portrait",          // 巻の既定
  entries:[{
    title, tags:[…],
    parts:[ {l:"品質", t:"masterpiece, best quality, …"},
            {l:"構図", t:"1girl, solo, upper body, …"}, … ],
    neg?:"…/キー上書き", set?:"…/キー上書き"
  }]
}]
// 完成版ポジ = parts.map(t).join(", ")
// 選択分     = checkedParts.map(t).join(", ")
```

ネガティブ専用巻（第十巻）は `parts` を持たず、ネガ断片そのものを本文として提示・コピーする。

## 9. 受入基準（実装後に検証）

1. どの頁も「完成版を写す」で `ポジ / Negative prompt: / 推奨設定` が揃ってコピーされる。
2. 構成要素のチェックを変えると組み立てプレビューが即時更新され、「選択分を写す」がその内容を返す。
3. 「全選択／全解除」が機能する。
4. 本・栞・検索のどの画面からも1アクションで本棚へ戻れる。パンくず表示が現在地と一致。
5. ブラウザ戻る/進む・Esc で破綻なく遷移。
6. 320〜1280px で横スクロールが発生せず、背表紙・棚板・カードが崩れない。
7. 栞・テーマがリロード後も保持される。
8. JS構文チェックとロジックテスト（完成版結合・選択結合・キー解決）が通過。

## 10. 追補（v3）完成アセット枠と 個別／統合 切替

### 10-1. 目的
完成プロンプトを「見本イラストとセット」で見せられるよう、各頁に**完成アセット枠**（プロンプト＋参考画像）を設ける。画像枠は空のプレースホルダから**アップロード**で埋められる。

### 10-2. 個別版／統合版の使い分け（各頁にタブ）
- 各頁の本文先頭に `[個別版] [統合版]` のタブを置く。
- **個別版**: 1プロンプト＝1枚の完成イラスト。構成要素（parts）＋完成版＋見本枠（1枚）。
- **統合版**: 複数の構図を1枚のシートにまとめるプロンプト。個別の要素に「シート指定」（`multiple views, reference sheet, 2x2 grid …`）を加えた完成版＋見本枠（1枚）。
- ネガ・推奨設定もモードで切替（統合版はシート向けサイズ `1216x1216`、`inconsistent design` をネガに追加）。
- 構成要素チェック・完成版コピーは両モードで機能。チェック状態と見本画像はモードごとに独立保持。

### 10-3. 見本画像枠（アップロード）
- 空枠: 破線フレーム＋「＋ 参考イラストをアップロード」。クリックでファイル選択。
- 取り込み時は最大1024pxへ縮小（JPEG化）して保存。
- 保存先: ブラウザ内 **IndexedDB**（`shelf_assets`）。キー＝`巻#頁|モード`。**各枠1枚**、差し替え・削除可。リロード後も表示。
- IndexedDB不可の環境ではメモリ保持にフォールバック（セッション内のみ）。

### 10-4. 追加の受入基準
9. 各頁にタブがあり、切替で完成版/ネガ/設定/構成要素が個別版↔統合版で入れ替わる。
10. 空枠から画像をアップロードでき、リロード後も同じ枠に表示される（IndexedDB保存）。
11. モードごとに見本画像とチェック状態が独立して保持される。
12. 画像の差し替え・削除ができる。

### 10-5. 旧スコープ外からの変更
- 「画像のアップロード」は本追補で**対象に変更**（各枠1枚・ローカル保存）。
- プロンプトのAI自動生成・翻訳、参照元/出典の表示は引き続きスコープ外。
